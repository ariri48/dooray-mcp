## 📢 기존 사용자: 업데이트 방법

### 🔄 이제 자동으로 업데이트됩니다

v2.1부터 **Claude Code를 시작할 때마다 서버가 스스로 최신 코드를 받아옵니다** (`git pull` 자동 실행).
평소에는 아무것도 할 필요 없이 Claude Code만 쓰면 됩니다.

- 인터넷이 안 되거나 로컬 수정이 있으면 업데이트만 건너뛰고 기존 코드로 정상 동작합니다.
- 자동 업데이트를 끄려면 `.env`에 `DOORAY_MCP_AUTO_UPDATE=false`를 추가하세요.

### ⚠️ 단, 이번 한 번만 수동 업데이트가 필요합니다

자동 업데이트 기능 자체를 받으려면 **마지막으로 한 번만** 아래를 실행하세요:

```bash
cd dooray-mcp        # 본인이 clone한 dooray-mcp 폴더로 이동
git pull
```

이후 Claude Code를 재시작하면 끝 — **앞으로는 재시작만 하면 항상 최신 버전**입니다.

> 💡 clone한 위치가 기억나지 않으면 터미널에서 `claude mcp list`를 실행하세요.
> dooray-mcp 항목에 등록된 `run_server.py` 경로가 곧 설치 위치입니다.

- ✅ **`.env`, `projects.json` 설정은 그대로 유지됩니다** — 다시 설정할 필요 없습니다.
- ✅ **의존성 추가가 없으므로 `pip install`도 필요 없습니다.**
- ✅ 기존 툴(태스크 조회/생성/수정 등)은 이름·동작 모두 그대로입니다.

### 업데이트 확인

Claude Code에서 아래처럼 물어보세요:

```
"두레이로 내 정보 조회해줘"        → dooray_common_me 툴이 실행되면 성공
"내 캘린더 목록 보여줘"            → dooray_calendar_list 툴이 실행되면 성공
```

## 🆕 신규 설치 (처음 쓰는 분)

### 사전 준비

| 항목 | 발급 방법 |
|------|-----------|
| Python 3.10+ | https://www.python.org/downloads/ |
| Claude Code | 설치 완료 상태 |
| Dooray API 토큰 | Dooray 웹 → 프로필 → **설정 → API 토큰**에서 개인 토큰 발급 |

### 설치

```bash
git clone https://github.com/ariri48/dooray-mcp.git
cd dooray-mcp
python setup.py     # 설치 마법사: 토큰 입력 → 의존성 설치 → MCP 등록까지 자동
```

설치 후 Claude Code를 재시작하면 바로 사용할 수 있습니다.

### 수동 설치 (마법사 없이)

```bash
pip install "mcp[cli]" requests python-dotenv
cp .env.example .env      # .env에 API 토큰, 테넌트 ID 입력
claude mcp add dooray-mcp -s user -- python /절대경로/dooray-mcp/run_server.py
```

## 🗂 v2.0.0에서 새로 할 수 있는 것

기존에는 **업무(태스크)** 기능만 지원했지만, 이제 Dooray 서비스 API 문서의 전체 영역을 사용할 수 있습니다.

| 서비스 | 할 수 있는 일 | 예시 요청 |
|--------|---------------|-----------|
| **업무 (기존)** | 태스크 조회/생성/수정/삭제, 댓글, 첨부파일, 검색, 일괄 처리 | "내 태스크 보여줘" |
| **업무 (확장)** | 완료 처리, 상태 변경, 상위 업무 지정, 담당자별 상태, 댓글 수정/삭제, 초안 | "태스크 12345 완료 처리해줘" |
| **프로젝트 관리** | 프로젝트 생성, 워크플로우/태그/마일스톤/템플릿 관리, 멤버 추가, 웹훅 등록 | "프로젝트에 '긴급' 태그 만들어줘" |
| **조직/멤버** | 조직 전체 멤버 검색, 내 정보 조회, 알림 스트림 | "김철수 멤버 ID 찾아줘" |
| **캘린더** | 일정 조회/생성/수정/삭제, 캘린더 관리 | "이번 주 내 일정 보여줘", "내일 3시에 회의 일정 잡아줘" |
| **드라이브** | 파일 목록/업로드/다운로드, 폴더 생성, 복사/이동, 공유 링크 | "드라이브에서 보고서.pdf 다운받아줘" |
| **위키** | 페이지 조회/작성/수정/이동, 댓글, 첨부파일 | "위키에 배포 가이드 페이지 만들어줘" |
| **메신저** | 1:1 메시지, 채널 메시지 전송/답장, 채널 생성/초대, 스레드 | "홍길동에게 '회의 10분 뒤 시작' 보내줘" |
| **예약** ⚠️ | 회의실 등 자원 검색, 예약 가능 시간 조회, 예약/취소 | "내일 오후 2~3시 예약 가능한 회의실 찾아서 잡아줘" |

> ⚠️ **예약(회의실) 기능 안내**: MCP에는 구현되어 있으나, 현재 우리 회사 두레이 테넌트에서
> 예약 서비스의 API 접근이 열려 있지 않아 `403 AUTH_FORBIDDEN_ACL_SERVICE_ERROR`가 발생합니다.
> 회의실 확인/예약은 두레이 웹/앱에서 직접 해주세요. (관리자가 Reservation API를 허용하면 즉시 사용 가능)
> 서비스별 API 허용 여부는 회사 관리자 설정에 따라 다를 수 있습니다.
| **주소록** | 연락처 목록/검색 | "주소록에서 삼성전자 담당자 찾아줘" |

## 🖥 공용 PC에서 사용하기 (토큰 저장 없이)

공용 PC에서는 토큰을 파일에 저장하지 않는 것이 안전합니다.
**`.env`의 `DOORAY_API_TOKEN`을 비워두면**, 세션에서 두레이 기능을 처음 사용할 때
토큰 입력 창이 자동으로 뜹니다. (macOS)

- 입력한 토큰은 **메모리에만 보관**되며 어떤 파일에도 저장되지 않습니다.
- **Claude Code를 종료하면 토큰이 사라지고**, 다음 세션에서 다시 물어봅니다.
- 입력 창에서 취소하면 해당 요청은 실패하며, 다음 요청 시 다시 입력할 수 있습니다.
- 개인 PC라면 기존처럼 `.env`에 토큰을 저장해 매번 입력을 생략할 수 있습니다.

## 🔒 안전장치

- **삭제 계열 기능**(태스크/페이지/파일/예약/일정 삭제 등)은 실수 방지를 위해
  Claude가 먼저 삭제 대상을 보여주고, `confirm=true`로 재호출해야 실제 삭제됩니다.
- API 토큰은 `.env` 파일에만 저장되며 절대 커밋되지 않습니다 (`.gitignore` 처리).
- 토큰 권한 = 본인 Dooray 계정 권한입니다. 본인이 접근할 수 없는 프로젝트/드라이브는 MCP로도 접근할 수 없습니다.

## 🔧 문제 해결 (FAQ)

**Q. 업데이트했는데 새 툴이 안 보여요.**
A. Claude Code를 "완전히" 종료했는지 확인하세요. 세션 유지 상태에서는 이전 서버가 계속 떠 있습니다. `git log --oneline -1`로 최신 커밋을 받았는지도 확인하세요.

**Q. `git pull`에서 충돌이 나요.**
A. 로컬에서 코드를 수정한 경우입니다. 수정 사항이 필요 없다면:
```bash
git stash && git pull
```

**Q. "DOORAY_API_TOKEN이 설정되지 않았습니다" 에러가 나요.**
A. dooray-mcp 폴더의 `.env` 파일에 토큰이 있는지 확인하세요. 없으면 `.env.example`을 복사해 작성하세요.

**Q. 특정 API 호출이 400/403으로 실패해요.**
A. ① 토큰 권한(프로젝트 멤버 여부) 확인 ② 관리자 전용 API인지 확인
③ `AUTH_FORBIDDEN_ACL_SERVICE_ERROR`라면 해당 **서비스의 API 접근이 테넌트(회사) 차원에서 막혀 있는 것**입니다
— MCP나 토큰 문제가 아니며, 두레이 관리자가 해당 서비스 API를 허용해야 사용할 수 있습니다 (현재 예약 서비스가 이 경우).
그래도 안 되면 `dooray_api_call` 툴로 API 문서의 파라미터를 그대로 지정해 호출할 수 있습니다:
```
"dooray_api_call로 GET /project/v1/projects를 member=me 파라미터로 호출해줘"
```

**Q. 예전처럼 업무 기능만 쓰고 싶어요.**
A. 그대로 쓰면 됩니다. 기존 툴은 이름/동작이 하나도 바뀌지 않았고, 새 툴은 요청할 때만 사용됩니다.
